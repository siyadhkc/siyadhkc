I specialize in **backend-heavy, security-conscious web systems** — the kind where architecture decisions matter and shortcuts break things in production.

My background runs deeper than most: before writing production Django, I spent two years deploying physical network infrastructure — CCTV systems, enterprise hardware, structured cabling. That's not a career detour. That's why I understand **latency, packet flow, and what happens below the application layer** when everyone else is guessing.

### 🧠 How I Think

Most developers think top-down: *request → view → response.*  
I think bottom-up: *hardware → network → OS → runtime → application → API contract.*

This matters when you're debugging a race condition in a Celery task at 2AM, or figuring out why your DRF serializer silently drops a field under load.
